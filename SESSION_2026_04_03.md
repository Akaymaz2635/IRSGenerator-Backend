### 3. Non Conformance Descriptions Page

**New backend endpoint:** `GET /api/inspections/{id}/nonconformance-descriptions`
File: `IRSGenerator.API/Controllers/InspectionsController.cs`

Returns:
```json
{
  "inspection_id": ..., "part_number": ..., "serial_number": ...,
  "operation_number": ..., "inspector": ..., "status": ...,
  "dimensional": [{ "item_no": "KN001", "description": "[KN001] 254 ± 0.01 (SHT01/D4) checks 238.99 U/MIN" }],
  "visual":      [{ "defect_id": 1, "index": 1, "description": "There is Dent D:0.1, W:0.1, L:0.13 see defect-1" }]
}
```

**Description format:**
- Dimensional: `[ITEM_NO] DIMENSION (BP_ZONE) checks VALUE U/MIN or VALUE O/T`
  - `U/MIN` = value below lower_limit
  - `O/T` = value above upper_limit
  - If attribute (no NumericPartResults): `[ITEM_NO] DIMENSION (BP_ZONE) — Attribute FAIL`
- Visual: `There is DEFECT_TYPE D:val, W:val, L:val (notes) see defect-N`
  - Short labels: D=Depth, W=Width, L=Length, R=Radius, A=Angle, H=Height
  - Only non-null/non-zero fields included

**New frontend page:** `IRSGenerator.API/wwwroot/static/js/pages/nonconformance-descriptions.js`
**Route:** `#/inspections/:id/ncr`
**Entry point:** "⚠ NC Descriptions" button in inspection detail page header (next to report buttons)

**Files modified:**
